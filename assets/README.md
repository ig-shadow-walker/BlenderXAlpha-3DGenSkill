# Assets

Media used by the project README.

## demo.gif (to be recorded)

The main README reserves a spot near the top for a demo. Record a short
screen capture and save it here as `demo.gif`, then activate it in the
README (see the commented-out image tag right under the title).

What makes a good demo for this skill:

- **Length:** 10 to 20 seconds. Trim the multi-minute generation wait; a
  caption like "generating (2 min)" or a jump cut is fine.
- **Show the whole arc:** the user's one-line scene prompt, the cost table
  and confirmation, then assets appearing and placing themselves in the
  Blender viewport.
- **Frame both windows** (Claude and Blender) if you can, so it's obvious
  the models land in a live Blender session.
- **Size:** aim for under ~10 MB so it loads quickly on GitHub. 1280px wide
  is plenty. An `.mp4` linked below the GIF is a nice extra for quality.

A still `demo.png` (hero screenshot of a finished scene) is a good
lightweight stand-in until the GIF exists.
